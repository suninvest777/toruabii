# Деплой на Render.com

## Проблема с портами

Если вы видите ошибку:
```
No open ports detected on 0.0.0.0
```

Это означает, что сервер слушает только на `localhost`, а Render требует, чтобы сервер слушал на `0.0.0.0`.

## Решение

### Вариант 1: Использовать server.js (рекомендуется)

Проект настроен для автоматической работы с Render. Просто убедитесь, что в настройках Render:

1. **Build Command**: `npm run build`
2. **Start Command**: `npm start` (или `node server.js`)

Скрипт `server.js` автоматически установит `HOST=0.0.0.0` перед запуском сервера.

### Вариант 2: Установить переменные окружения в Render

Если вы хотите установить переменные окружения вручную:

1. Перейдите в настройки вашего сервиса на Render
2. Откройте раздел "Environment"
3. Добавьте переменные:
   - `HOST` = `0.0.0.0`
   - `PORT` = (оставьте пустым, Render установит автоматически)

4. **Start Command**: `node dist/server/entry.mjs`

## Проверка

После деплоя проверьте логи. Вы должны увидеть:
```
[@astrojs/node] Server listening on http://0.0.0.0:10000
```

Вместо:
```
[@astrojs/node] Server listening on http://localhost:10000
```

## Дополнительная информация

- Render автоматически устанавливает переменную `PORT`
- Сервер должен слушать на `0.0.0.0`, чтобы быть доступным извне
- Команда `npm start` использует `server.js`, который автоматически настраивает правильный хост
